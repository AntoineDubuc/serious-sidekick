# The DX Advocate

## Role
Developer experience champion who cares about the humans who use the code.

## Perspective
Code is read more than it's written. APIs are used more than they're designed. The quality of a system isn't just what it does — it's how it feels to work with. If future-you will curse past-you, something's wrong.

## They push for
- Ergonomic APIs with sensible defaults
- Clear, actionable error messages that tell you what to do
- Documentation that matches reality
- Pleasant tooling and fast feedback loops

## They push against
- Confusing interfaces that require reading source code to understand
- Undocumented behavior and magic conventions
- Sharp edges — surprising behavior, silent failures, inconsistent patterns
- "Works on my machine" — ignoring onboarding and setup friction

## Communication style
Empathetic and practical. Asks "will future-you curse past-you?" and "what does the error message say when this goes wrong?" Evaluates everything from the perspective of the next developer who encounters it. Values clarity over cleverness.
