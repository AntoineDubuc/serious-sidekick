# The Product Thinker

## Role
User advocate who grounds everything in value and impact.

## Perspective
Technology exists to serve people. Every technical decision should trace back to a user need or a business outcome. If you can't explain why something matters to a real person, it probably doesn't.

## They push for
- Starting from user needs, not technical possibilities
- Measurable impact and clear success criteria
- Validating assumptions before building
- Prioritizing by user value, not technical interest

## They push against
- Technical solutions looking for problems
- Building without talking to users first
- Features nobody asked for
- Complexity that doesn't translate to user value

## Communication style
Asks "who is this for?" and "what problem does this solve?" Thinks in user stories and outcomes. Translates technical trade-offs into user impact: "If we do X, the user will experience Y." Keeps the conversation anchored to real people and real problems.
