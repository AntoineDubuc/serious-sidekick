Tutorial

Get started with
Serious Sidekick

You have Claude Code running in VS Code. Three prompts and you're set up.

3 prompts · 2 minutes
Step 1

Clone the repo

Paste this into Claude Code:

Claude Code
Clone https://github.com/AntoineDubuc/serious-sidekick to my home directory
Claude will run git clone and put the template at ~/serious-sidekick.
Step 2

Install the skills globally

Paste this into Claude Code:

Claude Code
Copy all skills from ~/serious-sidekick/.claude/skills/ to ~/.claude/skills/ so they're available globally in every project. Also copy the agents from ~/serious-sidekick/.claude/agents/ to ~/.claude/agents/
This installs all 23 skills and 5 Agent Teams agents into your global Claude Code config. They'll be available in every session, in every project.
!
That's the setup. You now have /serious-init available everywhere. The next step is using it.
Step 3

Set up any project

Open Claude Code in whatever project you're working on and paste:

Claude Code — your project
/serious-init
This copies into your project:
1

CLAUDE.md

Rules + feature index. Loaded every session, survives context compaction.

2

23 Skills

6 workflow skills + 17 feature cheat sheets that auto-load when topics come up.

3

38 Feature Docs

Deep-dive references with official docs, config options, and source URLs.

How it works

Three knowledge layers

Each loads at a different time to save context window space.

CLAUDE.md Always
Feature index + rules. Claude always knows what exists.
23 Skills On demand
Correct syntax and patterns. Auto-loads when topics come up.
38 Research Docs When needed
Full references with official docs, config, and URLs.
!
Why this matters: Claude hallucinates feature details from training data. These layers inject correct syntax, flags, and patterns exactly when needed.
Now what?

Pick a starting point

You don't need the full pipeline. Jump in anywhere.

A

Got an idea? Start talking.

/serious-conversation — explore it with personas before committing to anything.

B

Got a bug? Research it.

/serious-research — structured investigation with evidence grading.

C

Know what to build? Plan it.

/serious-plan — describe it in words, get a v6 implementation plan.

D

Have a plan? Build it.

/serious-code — TDD execution with 5 verification agents per task.

Done
